## Add `branch_error` + `branch_cost_usd` to the `query_search.branch` span
Files: observability/names.py, observability/cost_tracking.py, search_v2/streaming_orchestrator.py, observability_context/observability_architecture.md, observability_context/query_search_planning.md, docs/modules/api.md
Why: Two per-branch facts the /query_search trace plan called for were never wired — a per-branch soft-fail marker (previously only visible via the request-level failed-branch count + SSE payload + nested llm.generate ERROR, never on the branch span) and a per-branch cost rollup (only per-stage and per-request totals existed).
Approach:
- `branch_error` (str, soft-fail only): captured in the merge loop off the terminal `branch_results` payload (the same field api/main.py reads) and stamped on the branch span at close. A degradation — span stays UNSET, request verdict untouched (mirrors `trait_step_3_error`).
- `branch_cost_usd` (float, always-on, 0.0 for entity flows): a per-branch `RequestCostAccumulator` created beside each branch span and re-entered inside every one of the branch's tasks via a new `observability/cost_tracking.py::use_cost_scope` context manager. This was necessary because a standard branch's cost accrues across three SEPARATE sibling asyncio tasks (Step 2 / Step 3 / Stage 4) created by the merge loop, so context-snapshot-based `track_stage_cost()` can't span them; re-entering one shared accumulator object in each task sums correctly. Chose this over summing the per-stage `*_cost_usd` attributes (fragile; misses cost outside per-stage scopes). Both attrs written via a new `_finalize_and_end_branch_span` helper used at both span-close sites (merge loop + client-disconnect finally). Threaded the accumulator through `_run_under_span`, `_handle_finished_task`, `_handle_step2_done`. Covers both /query_search and /rerun_query_search (shared `_stream_from_branch_plan`).
Design context: names.py rules C/D/F (flat `branch_*` underscore leaves, `_usd` suffix, span-attr-only measures); the "span status vs request verdict are separate axes" convention (docs/conventions.md).
Testing notes: verified `use_cost_scope` semantics standalone (pre-branch cost → root only; in-branch → root + branch; per-branch isolation across create_task/gather). End-to-end Tempo verification pending (Bite 9): multi-branch query showing per-branch `branch_cost_usd` summing to ≈ request.cost_usd minus Step 0/1, and a forced soft-fail showing `branch_error` with the branch span status UNSET.
