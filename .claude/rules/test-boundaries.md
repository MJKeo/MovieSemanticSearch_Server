# Test File Boundaries

- Do NOT read, edit, or run test files unless explicitly asked
- Do NOT run the test suite unless explicitly asked
- If a command you ran happens to trigger tests and they fail,
  STOP and report the failures rather than investigating or fixing
- Test failures after implementation changes are expected and
  will be handled in a separate testing phase
- Never modify a test to make it pass; tests define expected
  behavior, not the other way around
