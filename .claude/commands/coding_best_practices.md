## Coding Best Practices (REQUIRED)

Your code MUST always adhere to the gold standard of industry best practices of the best software engineers unless explicitly specified otherwise. This means NO hacky code, always design in a way that sets future developers up to be as successful as possible while iterating on your code (generalizeable, modular).

Security is a requirement, not an option. Your code must be as secure as possible at all times. If you notice security vulnerabilities or potential security issues in the codebase you must attempt to fix them to the best of your abilities.

Efficiency is incredibly important. Write code such that it uses minimal memory and has minimal latency. This leads to a higher quality, more polished product.

Always consider every edge case while writing code. Catch exceptions when they could occur and handle them gracefully. Account for potential data corruption by verifying inputs and gracefully handling failures.

Your code is always being evaluated, put your best foot forward at all times.
